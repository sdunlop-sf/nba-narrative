🏗️
The Honest Assessment

Your data is harmonised. It's just not moving.

Denodo has done the hard work of federating data across your two SAP ERPs, your commerce platform, and Salesforce. But Denodo delivers point-in-time query results — it's a data access layer, not a profile engine. The data exists. It's just not connected to a rep visit brief, a portal recommendation, or a propensity score at the moment the customer is active. Data Cloud adds the real-time event stream, the unified customer profile, and the segment engine that makes that data actionable. They're complementary — Denodo for integration governance, Data Cloud for activation.

The Connected Stack
How the layers work together to deliver Smart Selling
01
Data Cloud
Unified Customer Profile

Ingests your 2 SAP ERPs, SAP Commerce, Salesforce CRM, and Denodo feeds into a single real-time customer profile. Replaces the point-in-time DataFizz layer with live segment membership — customer enters a distribution-gap segment the moment the signal fires, not the next reporting cycle.

02
Einstein Studio + Flow
NBA Engine

Replaces DataFizz's ~60% accuracy propensity models with Einstein Studio, trained on your actual CRM, ERP, and commerce data. Scores each customer against each available NBA — cross-category upsell, contract renewal, tap optimisation, range expansion. Engine picks the winner. Rep gets the brief.

03
Salesforce Personalisation
Rep CRM + Portal

Surfaces AsahiIQ visit briefs inside Salesforce for reps, and delivers real-time product recommendations in the SAP Commerce portal for customers. Consistent intelligence across both the human-led and digital channels. Requires only a JavaScript tag on the portal — no structural rebuild.

04
Marketing Cloud
Marketing on Core

Trigger-based promotional campaigns and segmented email journeys, fired by NBA signals. Contract approaching → targeted renewal offer. Distribution gap identified → rep prompt + customer email in parallel. Replaces scheduled batch campaigns with event-driven execution.

05
Salesforce Promotions
Trade Incentive Engine

Promotions define the rules — eligibility, mechanics, thresholds, redemption limits. When an NBA fires, the engine checks which promotions the customer qualifies for and surfaces the right offer through whichever channel executes it: rep brief, email, or portal. The offer is always backed by a live, governed promotion — not a manually authored one-off. Replaces trade incentives negotiated and tracked ad hoc with an auditable promotions layer that reps and Marketing Cloud both draw from.

06
Agentforce
"Ask the Agent" — Phase 2

Phase 2 capability: reps ask the Agentforce agent directly — "What should I pitch to The Paddington Arms this week?" — and get back a ranked, signal-backed answer. Reduces the 2 hrs/day admin burden to minutes. Agent handles call prep, account summary, and post-visit logging.

07
Feedback Loop
Every action retrains the model

Rep accepts an NBA → outcome logged → positive training signal → Einstein Studio retrains. Customer orders a recommended product on portal → signal captured → model improves. The engine gets smarter every time a rep or customer acts on a recommendation.

Master Flow
NBA Orchestration Engine — Smart Selling
1
Load

Query the Active NBA Catalogue

Pull all active NBAs — cross-category upsell actions, contract renewal prompts, range gap closures, tap optimisation tasks — where today falls within the valid window. This is the candidate pool for this outlet, this run.

No hardcoded actions. A new NBA for Asahi non-alc push is a new record in the catalogue. Retire it by flipping Status to Inactive. Zero code change required.
2
Loop

For each NBA — Eligibility → Score → Gate

Three subflows run in sequence for every action in the catalogue. None of them know anything about specific NBAs — they process whatever record is passed in.

Subflow loop — runs per NBA
Subflow A
Eligibility Check

Is this outlet in the eligible Data Cloud segment? Has this action been shown recently (frequency cap)? Any active exclusions (contract dispute, account on hold)?

→
Subflow B
Score

Call Einstein Studio — upgraded from DataFizz ~60% accuracy. Returns a propensity score for this outlet + action based on purchase history, distribution gaps, and postcode mix.

→
Subflow C
Threshold Gate

Score ≥ minimum propensity? Add to candidates weighted by score × margin value. Below threshold? Discard. Protects reps from low-confidence recommendations.

3–4
Decide

Arbitrate & Execute

Candidates sorted. Winner surfaced to the rep and/or portal.

Step 3 — Arbitrate

Rank the candidates

Sort by propensity × margin value. Apply channel rules (one action per channel per visit). Top 3 ranked actions win. Ties broken by commercial priority.

Step 4 — Execute

Surface the brief

Push ranked NBAs to Salesforce as the AsahiIQ pre-visit card. Trigger portal recommendation update. Fire Marketing Cloud trigger if digital channel is the execution path. Log decision for attribution.

Feedback Loop

On DataFizz vs. Einstein Studio

~60% accuracy is a coin flip with extra steps

DataFizz propensity models work from static data exports. Einstein Studio trains on live Data Cloud segments — updated as customers purchase, browse the portal, and respond to rep visits. The accuracy gap closes because the model sees the actual signal, not a monthly extract of it.

On the SAP Commerce integration

The portal launch in Aug/Sep is the moment

EY is implementing the SAP Commerce Cloud portal redesign. The personalisation layer requires one JavaScript tag in the site header — no structural changes to the SAP build. The right time to activate it is at portal launch, not after. Miss that window and you're retro-fitting.
