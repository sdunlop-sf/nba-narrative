02
The Value Function
A composable scoring formula where each component maps to a primitive. The weights are configurable — industry templates ship different weightings for different business strategies.
Scoring Formula
weighted = propensity × expectedValue × pclvMultiplier × priority × adaptiveWeight
Propensity — ML model prediction. How likely is this customer to respond positively? From Einstein Studio.
Expected Value — Business value of the action if accepted. From the action catalogue. Optionally from RTOM margin data.
pCLV Multiplier — Customer's predicted lifetime value, normalized against the median. High-value customers get priority. From Data Cloud CIs.
Priority — Business-assigned urgency weight. Retention outranks upsell. From the action catalogue.
Adaptive Weight — Historical win rate. Actions that produce outcomes get boosted. This is the learning loop in the formula.
Weighted Score — The composite output. Determines the ranking. The value function IS the business strategy, expressed as math.
Why this matters for templates
A banking template ships with high priority on compliance-tagged actions and a pCLV multiplier calibrated for retail banking value bands. A telco template ships with churn-weighted propensity and higher adaptive weight sensitivity (faster learning from engagement signals). The formula is the same. The weights define the industry.

03
Tiered Reasoning
The reasoning engine isn't one thing. It's a confidence-gated escalation from fast deterministic scoring to progressively more expensive reasoning. Most decisions never need LLM involvement.
Tier 1
~80%
⚡
Deterministic Scoring
Clear winner by formula. Wide gap between top-ranked and second-ranked action. No ambiguity, no LLM call. Fastest, cheapest, most auditable. The formula is the decision.
↓
Gap ≤ threshold → escalate
Tier 1.5
~15%
🧠
Agentic Re-rank
Narrow gap between top actions. LLM receives the shortlist plus relationship context and re-ranks with reasoning. Reports confidence. If low confidence → escalates to Tier 2.
↓
Low confidence or narrow proximity → escalate
Tier 2
~4%
👤
Human Escalation
System confidence too low for autonomous action. Decision queued for human review with full context, reasoning trace, and recommended action. Slack integration for approval workflows.
↓
No eligible action meets threshold → compose
Compose
~1%
✨
Agentic Composition
Nothing in the catalogue fits. LLM composes a novel treatment within constitutional rails — budget caps, policy constraints, confidence thresholds. Created treatment enters review queue before execution.
The cost curve
Tier 1 is sub-millisecond, zero LLM cost. Tier 1.5 is ~200ms, one LLM call. Tier 2 is human-time. Composition is one LLM call + human review. The architecture is designed so that the expensive path only fires when the cheap path genuinely can't decide. As the learning loop improves model accuracy and adaptive weights, more decisions resolve at Tier 1 over time.

The pre-compute pattern
The Hub runs batch on its own schedule, writes latent decisions to Data Cloud, and the serving layer reads from Data Cloud at whatever concurrency it needs. No Apex in the hot path. No governor limits in the serving layer. SP without the Hub serves offers based on segments and rules. SP with the Hub serves offers backed by compounding relationship intelligence. That's the upgrade story.
05
Outcome Attribution
"Accepted" is a proxy. The learning loop needs to know what actually happened — did the customer renew, buy, stay?
Signal 1 — Immediate
Decision Response
Agent or system accepts/rejects the recommendation. CampaignMember status writes back to Data Cloud. Available today.
Signal 2 — Near-term
Business Outcome
Match NBADecision__c to downstream CRM events (Opportunity closed, Case resolved, renewal completed) within a configurable attribution window per module.
Signal 3 — Long-term
Relationship Trajectory
Did the sequence of decisions move pCLV up or down? Did churn risk decrease? The decision journal reveals which patterns compound value over months.
Signal quality determines loop quality
If "accepted" is the training signal, the system learns to optimise for agent agreement. If "renewed within 90 days" is the signal, it learns to optimise for business outcomes. The quality of the learning loop — and every future decision — depends on closing the gap between proxy metrics and real attribution.

The orchestration layer
None of these platform capabilities are new. What's new is the orchestration: the system that reads relationship state, constrains the action space, applies the value function, gates reasoning by confidence, records the decision, and feeds outcomes back into the loop. The accelerator doesn't replace platform capabilities. It connects them into a system that compounds.

The switching cost inversion
Traditional vendor lock-in is technical — proprietary APIs, data formats, integrations. Memory-first lock-in is institutional. The customer's data, decisions, outcomes, and learned weights are the asset. The switching cost isn't learning new software. It's losing everything the system has learned about your customers. This is a fundamentally different value proposition: the system becomes more valuable the longer you use it, not because of vendor lock-in, but because of accumulated intelligence.
08
Why This Wasn't Possible Before
Three platform capabilities that didn't exist (or weren't mature) even 18 months ago make this architecture viable now.
☁️
Data Cloud CIs
Calculated Insights turn CRM + behavioral data into queryable signals (pCLV, engagement scores, churn probability). This is relationship state as infrastructure. Without it, every decision requires re-computing customer context from raw data.
🤖
Agentforce + LLMs
Reasoning capability within the platform at sub-second latency. The pluggable brain layer becomes practical when you can call an LLM in 200ms. Makes tiered reasoning viable — deterministic by default, LLM when needed.
🎯
Einstein Studio
Train, deploy, and retrain ML models on Data Cloud data without leaving the platform. Closes the learning loop: outcomes feed back into model training, which feeds back into the value function, which improves the next decision.
🔀

The timing thesis: The platform pieces exist but nobody has connected them into a compounding system. This space was historically ceded to specialist vendors who built on pre-cloud, pre-AI infrastructure. The opportunity is to build the orchestration layer that turns platform capabilities into memory-first decisioning — before anyone else does.
