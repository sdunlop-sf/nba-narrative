Data Transformation Summary

How data is transformed at each stage of the pipeline.

StageInputTransformationOutput
Scoring (1-3 → 4) Contact ID, segment memberships Einstein Studio model inference Propensity score (0.00–1.00) per outcome type
Arbitration (4) Scores, module catalogue, eligibility rules Sequential filter chain with deterministic elimination Ranked list of eligible actions (may be empty)
Composition (7-8) Full decision context: scores, segments, dropped actions with rejection reasons LLM generates novel treatment grounded in decision context NBA_Composed_Action__c record with channel, offer, content, rationale
Rails Validation (9) Composed treatment marshalled to standard data structure Identical Apex validation methods as catalogued actions Pass (all rails) or reject (any rail fails) + JSON audit trail
Confidence Gate (12) Rails-validated composition with agent confidence score and signal analysis When auto-approve is enabled: evaluate confidence threshold and check for conflicting signals. When off: route all to human review. Tier 1: auto-approved for execution (auto-approve on, high confidence, no conflicts). Tier 2: routed to human approval queue.
Human Approval (13) Tier 2 composition with full context and conflict details Human judgment: approve, reject, or edit Approved treatment ready for execution
Outcome Capture (15) Customer response to executed treatment Response mapped to outcome event, published to Einstein Studio Training signal for model retraining + promotion candidacy
Promotion (17) Composed action with positive outcome evidence Derive score threshold, eligibility rules, priority weight from composition context + outcomes New NBAAction__c catalogue entry — participates in future arbitration
